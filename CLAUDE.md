# Open Intranet Docs — CLAUDE.md

Documentation site for [Open Intranet](https://www.open-intranet.com), built with Astro Starlight.

## Quick reference

- **Public URL**: `https://www.open-intranet.com/docs/` (canonical, served via reverse proxy)
- **Direct URL**: `https://docs.open-intranet.com/docs/` (custom domain on CF Worker, redirects from `/`)
- **GitHub repo**: `droptica/open-intranet-docs` (public)
- **Main site repo**: `droptica/open-intranet-com` (private) — marketing website
- **Product repo**: `droptica/open-intranet` — the actual Drupal intranet
- **Dev server**: `npm run dev` → `localhost:4321/docs/`
- **Build**: `npm run build` → `dist/`
- **Deploy**: Cloudflare Workers (auto-build from GitHub on push to `main`)

## Tech stack

| Layer | Choice |
|-------|--------|
| Framework | Astro 5.x + Starlight |
| Search | Pagefind (built-in with Starlight) |
| Deploy | Cloudflare Workers (static assets + worker script) |

## Architecture — reverse proxy setup

The docs are served at `open-intranet.com/docs/*` via a Cloudflare Worker reverse proxy:

1. **Astro config** has `base: '/docs/'` — all generated links include `/docs/` prefix
2. **Build output** goes to `dist/` (files are NOT in `dist/docs/`, Astro only prefixes URLs)
3. **Worker script** (`src/worker.js`) strips `/docs` from the request path before fetching from static assets
4. **CF Route** `www.open-intranet.com/docs/*` → `open-intranet-docs` Worker
5. **Custom domain** `docs.open-intranet.com` hides the `*.workers.dev` URL

Flow: `open-intranet.com/docs/getting-started/installation/`
→ Worker receives `/docs/getting-started/installation/`
→ Strips to `/getting-started/installation/`
→ Serves `dist/getting-started/installation/index.html`

## SEO

- `public/robots.txt` has `Disallow: /` — blocks indexing on `*.workers.dev` and `docs.open-intranet.com`
- Canonical indexing happens only via `www.open-intranet.com/docs/` (main domain)

## Starlight customizations

- **SiteTitle component** (`src/components/SiteTitle.astro`) — overrides default so logo links to `https://www.open-intranet.com` (main site) instead of docs root
- **Edit links** — point to `https://github.com/droptica/open-intranet-docs/edit/main/`

## Key files

- `astro.config.mjs` — Starlight config, sidebar, base path
- `src/worker.js` — CF Worker that strips `/docs` prefix and handles redirects
- `wrangler.json` — CF Worker deployment config with ASSETS binding

## Known gotchas

- `base: '/docs/'` only affects generated URLs, NOT the `dist/` directory structure
- The worker script is required because CF serves from `dist/` root but requests come with `/docs/` prefix
- `wrangler.json` must have `"binding": "ASSETS"` for the worker script to access static files
- CF Workers Builds sometimes doesn't trigger on push — an empty commit can force a rebuild
- The worker was deployed to `devops@droptica` CF account (not personal) — `wrangler deploy` from CLI requires login to that account
